AI Art Critic - Data Preparation

This report explains how the data was prepared before training the model. The goal was to make sure the data was correct, clean, and ready to use.

The model learns to predict art styles from images (for example Impressionism, Cubism, Realism, Baroque, and other styles in the dataset).

The steps below are shown in the same order they were done. For each step, it shows what was done and what the result was.

1) Quick Map: What was done

Workstream Main data preparation work Output / result
Local image dataset build Loaded the ArtBench file, checked needed columns, downloaded images into local style folders, and saved cleaned CSV files. Local image folders, a full updated CSV, a simple training CSV, and a failure CSV.
Cross-dataset path linking Matched ArtEmis painting names to real WikiArt image paths by scanning image files and matching names. A new CSV with added painting path columns.
Classification input preparation Loaded train/validation CSVs, mapped labels to style names, removed broken rows, and applied image preprocessing settings. Clean train/validation data ready for training.

ArtEmis - matching names to WikiArt paths
# Build map: image stem -> possible relative paths
name_to_paths = {}
for img_path in image_files:
    stem = img_path.stem.strip().lower()
    rel_to_datasets = img_path.relative_to(project_root / "datasets")
    name_to_paths.setdefault(stem, []).append(rel_to_datasets.as_posix())

def resolve_image_path(name: str):
    if pd.isna(name):
        return None
    key = str(name).strip().lower()
    matches = name_to_paths.get(key, [])
    if not matches:
        return None
    return sorted(matches)[0]  # deterministic

df["painting_path"] = df["painting"].apply(resolve_image_path)

4) Step-by-step: WikiArt preparation in experiment notebooks

In the experiment notebooks, the main goal was to keep training input clean and stable. The same preparation flow was reused across tests.

Each image is linked to a numeric label that points to an art style class. This is how the model learns which style each painting belongs to.

  1. Automatically found the dataset root and train/validation CSV files.
  2. Built a clear label-to-style mapping from folder names.
  3. Removed rows with missing files and, in later tests, checked if images were actually readable.
  4. Applied image processing (resize, normalize, and extra image changes during training).
  5. Used stable dataloader settings for Windows so loading did not fail or stall.
WikiArt - finding files + label mapping
def find_split_csv(data_dir: Path, split: str) -> Path:
    split = split.lower()
    csvs = sorted(data_dir.glob("*.csv"))
    preferred = [p for p in csvs if split in p.stem.lower() and "style" in p.stem.lower()]
    if preferred:
        return preferred[0]
    fallback = [p for p in csvs if split in p.stem.lower()]
    if fallback:
        return fallback[0]
    raise FileNotFoundError(f"Could not find a '{split}' CSV in {data_dir}")

def filter_existing_rows(df: pd.DataFrame, image_root: Path, split_name: str) -> pd.DataFrame:
    full_paths = df["relative_path"].map(lambda p: image_root / p)
    exists_mask = full_paths.map(lambda p: p.exists())
    kept = int(exists_mask.sum())
    removed = int((~exists_mask).sum())
    print(f"{split_name}: kept {kept:,}, removed {removed:,} missing files")
    return df.loc[exists_mask].reset_index(drop=True)

5) How the preparation improved over time

  • Early runs: basic loading, label mapping, missing-file cleanup, and simple transforms.
  • Middle runs: stronger image changes during training and better checks for unreadable images.
  • Later runs: faster filtering, safer checks, and more stable loading settings.
  • Recovery stage: the same preparation structure was kept, and saved results could still be used.

6) Final outputs created by preparation

  • ArtBench: saved a new CSV with local image paths, a simple CSV for training, a failed-download CSV, and local image folders.
  • ArtEmis: saved a new CSV with matched WikiArt image paths.
  • WikiArt: kept clean tables and stable data loaders ready for training.

Final takeaway: the data was not just collected. It was checked, fixed, and cleaned before training. That helped training run better and gave more reliable results.
